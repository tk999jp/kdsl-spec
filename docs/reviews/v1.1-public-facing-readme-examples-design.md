## 1. Purpose

Define the minimum public-facing documentation set for a future release candidate without approving publication, release, tag creation, tag movement, or Release Assets operations.

This document is design only. It does not create public approval, release approval, tag approval, repository visibility approval, or announcement approval.

## 2. Public-facing README goals

```text
explain what KDSL is
explain what R1 / KDSL_RESULT is
explain KDSL / KDSL-DP / ADPS boundary
explain safety gates and protected terms
explain validator scope and non-scope
link to core specs, profiles, lint, bridge, templates, examples, and validator tools
state draft maturity clearly
state no release/public readiness unless explicitly approved
```
