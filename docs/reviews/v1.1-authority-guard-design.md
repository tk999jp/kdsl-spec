# KDSL v1.1 Authority Guard Design

status: design-draft
scope: KDSL_RESULT NEXT and COMMIT fields
release: none
public: not_yet
implementation: not_started

## 1. Purpose

Define a conservative validator design for keeping KDSL_RESULT NEXT and COMMIT as result/report metadata, not as authority transfer.

This document is design only. It does not create release approval, public approval, tag approval, commit approval, or next-task approval.

## 2. Field intent

```text
NEXT := proposal-only field
COMMIT := actual/proposed/none record field
```

## 3. Allowed shapes

### NEXT

```text
NEXT: none
NEXT: proposed:<text>
NEXT: proposal:<text>
NEXT: 提案:<text>
NEXT: 次候補:<text>
```

### COMMIT

```text
COMMIT: actual:none / proposed:none
COMMIT: actual:<message> / proposed:none
COMMIT: actual:none / proposed:<message>
COMMIT: actual:<message> / proposed:<message>
```

## 4. Warn conditions

```text
NEXT field exists but proposal-only marker is absent
COMMIT field exists but actual/proposed/none separation is absent
COMMIT field contains only a bare message
NEXT or COMMIT field is multiline and cannot be parsed by simple field reader
```

## 5. Fail conditions

```text
NEXT is clearly phrased as authority transfer rather than proposal metadata
COMMIT is clearly phrased as authority transfer rather than actual/proposed metadata
NEXT or COMMIT is missing in KDSL_RESULT when R1 required-block validation is expected
```

## 6. Non-goals

```text
No natural language complete semantic parser
No approval judgment
No runtime judgment
No release judgment
No publicization judgment
No Git operation execution
```

## 7. Recommended implementation slice

```text
script: tools/validator/r1_authority_guard.py
input: KDSL_RESULT text
output: VALIDATION_RESULT
exit: 0 pass / 1 warn / 2 fail
mode: allow-list first, minimal fail rules second
```

## 8. Safety boundaries

```text
validator pass != U approval
validator pass != implementation validity
validator pass != RT:v
NEXT validation != next-task approval
COMMIT validation != commit approval
```
