```text

  KDSL本体の定義 / operator / mode / safety / 保護語

  dev-prompt / converter の運用仕様

  KDSL_RESULT / Evidence / Runtime verification / Authority separation
```

## 3. Alignment result

### Core

OK:

```text
KDSLはLLM直投入可能な半構造化prompt記法として定義されている
安全gate保持がtoken削減より優先されている
KDSL-DPはP1/P1L正規化まで実行不可とされている
RT:v / NEXT / COMMIT条件がCore側にも現れている
```

### R1

OK:

```text
R1は単なる報告テンプレではなく検収票として定義されている
KDSL_RESULT blockが明確
RT:vの有効根拠/無効根拠が分離されている
EVIDENCEでOBSERVED/INFERRED/NOT_OBSERVED/UNVERIFIEDが分離されている
AUTHORITYでcommit/push/releaseが分離されている
```

### Templates

OK:

```text
base/result/taskに責務分離されている
template未読時停止が明示されている
instance側でPhase/HEAD/target/evidence/authorityを明示する構造になっている
```

### Examples

OK:

```text
before/after/R1に分かれており、テンプレート化の効果が確認できる
Examplesは正本ではないと明記されている
```

### Validator design

OK:

```text
実装ではなく設計扱い
validator pass != RT:v / U承認 / 要件妥当性 と明記
R1 validatorとtemplate lintが分離されている
```
