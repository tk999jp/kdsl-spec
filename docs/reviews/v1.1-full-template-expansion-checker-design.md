# KDSL v1.1 Full Template Expansion Checker Design

status: design-draft
scope: KDSL_PROMPT template expansion / explicit inheritance checks
release: none
public: not_yet
implementation: not_started

## 1. Purpose

Define a validator design for checking whether referenced KDSL templates are either expanded into the target prompt or explicitly inherited with readable source references.

This document is design only. It does not approve release, publicization, tag operations, Release Assets, implementation correctness, or runtime verification.

## 2. Problem

The existing template reference checker detects known template paths and required safety gates. It does not confirm whether the referenced template content has been expanded, read, or explicitly inherited.

A full expansion checker should reduce these risks:

```text
referenced template not read
referenced template assumed from memory
template alias inferred without source
template required sections missing after compression
template inheritance unclear
```

## 3. Input

```text
KDSL_PROMPT text
optional repository root
optional known template index
optional expansion evidence markers
```

## 4. Known template groups

```text
base:
  templates/base/kdsl_base_dev.md

result:
  templates/result/r1_result_spec.md

task:
  templates/tasks/task_docs_state_closeout.md
  templates/tasks/task_corrective_impl.md
  templates/tasks/task_investigation_only.md
```

## 5. Expansion evidence markers

A referenced template is treated as expanded or explicitly inherited only if at least one acceptable marker exists.

```text
expanded_from:<template path>
inherits:<template path>
template_read:<template path>
source_template:<template path>
```

## 6. Required section groups

### base template minimum sections

```text
format/profile/mode/safety
目的
AUTHORITY
禁止

```

### result template minimum sections


### task template minimum sections

```text

```

## 7. Pass conditions

```text
no template reference exists
or every referenced template has an expansion/inheritance marker
and required section group for the referenced template family is present
```

## 8. Warn conditions

```text
template reference exists and expansion marker exists, but required sections are only partially present
multiple templates are referenced and one family cannot be classified
section wording is present but ambiguous
template inheritance is explicit but expansion evidence is not recorded
```

## 9. Fail conditions

```text
template reference exists but no expansion/inheritance marker exists
unknown template path or alias is used
known template family is referenced but required section group is absent
KDSL-DP is treated as directly executable before P1/P1L normalization
```

## 10. Non-goals

```text
No full semantic equivalence proof
No natural language complete parser
No runtime verification
No user approval judgment
No release/public/tag/Release Assets judgment
No automatic template content insertion
```

## 11. Recommended implementation slice

```text
script: tools/validator/kdsl_template_expansion.py
input: KDSL_PROMPT text
output: VALIDATION_RESULT
exit: 0 pass / 1 warn / 2 fail
mode: known-path only, no alias inference
```

## 12. Safety boundaries

```text
validator pass != template semantic equivalence
validator pass != U approval
validator pass != implementation validity
validator pass != RT:v
validator pass != release readiness
unknown template/profile/alias/preset must not be inferred
```
